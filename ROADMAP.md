# Roadmap

Ideas explicitly out of scope for `foldr` 0.1.0:

- Multi-planet iterative masking (search, mask, re-search)
- GP / `wotan`-based detrending (only running-median detrending is supported)
- Target pixel file / centroid analysis
- Vetting diagnostics (odd-even depth test, secondary eclipse search) — likely
  a future standalone tool, `vetr`
- MAST or other network downloads — `foldr` only reads local files
- River plots
- TTV (transit timing variation) analysis
- Interactive or GUI modes
